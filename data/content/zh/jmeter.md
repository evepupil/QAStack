# Apache JMeter

Apache JMeter 是一个开源的性能测试工具，专为负载测试和性能测量而设计。它支持多种协议，是企业级性能测试的首选工具之一。

## 主要特点

- **多协议支持** - HTTP、HTTPS、FTP、JDBC、SOAP、REST 等
- **图形化界面** - 直观的测试计划设计
- **分布式测试** - 支持多机器负载生成
- **丰富的报告** - 详细的性能指标和图表
- **可扩展性** - 支持插件和自定义组件
- **录制功能** - 可录制浏览器操作
- **参数化** - 支持数据驱动测试

## 使用场景

- Web 应用性能测试
- API 负载测试
- 数据库性能测试
- FTP 服务器测试
- 压力测试和容量规划
- 性能基准测试

## 快速开始

### 安装

1. 下载 JMeter（需要 Java 8+）
2. 解压并运行 `jmeter.bat`（Windows）或 `jmeter.sh`（Linux/Mac）

### 基础用法

1. **创建测试计划**
   - 添加线程组（模拟用户）
   - 添加 HTTP 请求采样器
   - 添加监听器（查看结果）

2. **配置线程组**
   - 线程数：模拟的用户数
   - Ramp-up 时间：启动所有线程的时间
   - 循环次数：每个线程执行的次数

3. **添加断言**
   - 响应断言
   - 持续时间断言
   - JSON 断言

4. **运行测试**
   - 点击绿色启动按钮
   - 查看实时结果
   - 生成测试报告

## 优势

- ✅ **完全免费** - 无任何使用限制
- ✅ **功能全面** - 支持多种测试场景
- ✅ **成熟稳定** - 经过多年验证
- ✅ **社区活跃** - 丰富的插件和资源
- ✅ **跨平台** - 支持所有主流操作系统

## 局限性

- ⚠️ **学习曲线** - 界面复杂，需要时间学习
- ⚠️ **资源消耗** - 大规模测试需要较多资源
- ⚠️ **GUI 性能** - 图形界面在高负载下可能卡顿
- ⚠️ **现代化程度** - 界面相对传统

## 定价

Apache JMeter 完全**免费开源**，采用 Apache 2.0 许可证。

## 最佳实践

### 命令行模式

对于大规模测试，建议使用命令行模式：

```bash
jmeter -n -t test_plan.jmx -l results.jtl -e -o report_folder
```

### 分布式测试

1. 配置主控机和从属机
2. 在主控机上启动测试
3. 从属机自动执行负载生成

### 性能优化

- 使用非 GUI 模式运行测试
- 禁用不必要的监听器
- 使用 CSV 数据集进行参数化
- 合理配置线程数和 Ramp-up 时间
